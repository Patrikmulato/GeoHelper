---
description: Backend rules for backend/src/** files
globs: backend/src/**
---

- Test runner is **Node.js native (`node:test`)** — never use Jest; import `describe`/`it` from `node:test` and `assert` from `node:assert/strict`
- DTO validation uses class-validator decorators; `ValidationPipe` is configured with `whitelist: true, forbidNonWhitelisted: true` — unknown fields return 400
- If adding a new filter value or filter type, update `dto/filter-request.dto.ts` **and** `src/types/map-data.ts` in the frontend together
- `geo-car-helpdesk.ts` is auto-generated by `sync:data` — hand edits will be overwritten on next sync
- Tooltip HTML is built server-side in `data.service.ts` and delivered to the frontend as a string — do not move this to the frontend
- CORS is configured in `app-setup.ts`; allowed origins are `localhost:3000`, `CORS_ORIGIN` env var, and `FRONTEND_URL` env var
- The filter cache in `data.service.ts` is in-memory with 300-entry FIFO eviction — no TTL, no persistence
